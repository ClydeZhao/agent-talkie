## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 浏览器 → relay WebSocket | 不可信 JSON；必须由 Zod / safeParseEnvelope 校验后再进入应用逻辑 |
| sessionStorage | 仅本标签页；凭证可被同源脚本读取，依赖 localhost 信任模型（v2.0） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | T | `dispatchPostHandshake` | mitigate | 所有入站 JSON 使用 `@agent-talkie/protocol` 与本地 `transcriptCatchupMessageSchema` 解析；拒绝则丢弃帧，不 `eval` envelope |
| T-07-01-02 | I | `session.register` / `resume` | accept | localhost 下 `isHuman` 由客户端声明，与现有 relay 一致；不引入远程认证（RSEC 延期） |
| T-07-01-03 | D | 大型 JSON 帧 | mitigate | 不向 relay 发送超过合理大小的 payload；依赖 relay `MAX_INBOUND_WS_BYTES` 对入站限制 |
</threat_model>
