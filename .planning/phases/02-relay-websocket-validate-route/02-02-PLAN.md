- `SESSION_RESUME_TTL_MS = 604800000`
- `RECONNECT_PEPPER = process.env.AGENT_TALKIE_RECONNECT_PEPPER ?? 'dev-reconnect-pepper'`

**Handshake:** First client→relay text message JSON must satisfy `relayClientHandshakeSchema` (see Task 1). Relay responds with either:
- `{"type":"handshake.ack","negotiatedVersion":<n>,"relay":{"minVersion":1,"maxVersion":1}}` where `negotiatedVersion === agreeProtocolVersion(client, RELAY_SUPPORTED_VERSIONS)`, or
- `{"type":"handshake.nack",...}` where payload equals `buildVersionMismatchFailure(RELAY_SUPPORTED_VERSIONS)` fields plus `type: 'handshake.nack'` wrapper object: `{ type: 'handshake.nack', error: 'version_mismatch', relay: { minVersion, maxVersion }, message: string }` using the same nested `relay` object as `buildVersionMismatchFailure`.

**Session bind messages (after handshake.ack, before envelopes):**
- New session: `{ "type": "session.register", "newSession": { "displayName": string, "runtime": string, "workspaceLabel": string, "branch"?: string, "focus"?: string } }` → relay calls `createSession(db, input)` from `@agent-talkie/persistence`, generates reconnect secret 32 bytes `crypto.randomBytes(32).toString('base64url')`, sets `reconnect_secret_hash` + `reconnect_valid_until = Date.now() + SESSION_RESUME_TTL_MS` via prepared SQL `UPDATE sessions SET reconnect_secret_hash=?, reconnect_valid_until=? WHERE id=?`, responds `{ "type":"session.registered","sessionId":"<uuid>","reconnectSecret":"<base64url>","displayName":"..." }`.
- Resume: `{ "type": "session.resume", "sessionId": "<uuid v7>", "reconnectSecret": "<base64url>" }` → constant-time hash compare; if ok `UPDATE` new hash + new `reconnect_valid_until`, respond `{ "type":"session.resumed","sessionId":"..."}`; if fail `{ "type":"protocol.error","error":"resume_rejected"}` and close.

**Duplicate session connection policy:** If a new socket completes `session.resume` or `session.register` for `sessionId` S while another open socket is already bound to S, call `.close()` on the older `WebSocket` first, then bind the new one (second connection wins).

**Inbound envelope path (after bound):** Parse JSON; if not object, send `{ "type":"protocol.error","error":"invalid_json"}`; if object, run `safeParseEnvelope`; on failure send `{ "type":"protocol.error","error":"invalid_envelope","issues":[...] }` where `issues` equals `formatEnvelopeIssues(result).issues` (array of `{path:string,message:string}`); on success call a stub `dispatchValidatedEnvelope(wsContext, envelope)` exported from `server.ts` that currently only returns without DB writes (routing in 02-03).

**Dependencies:** `packages/relay/package.json` must include `"ws": "^8.20.0"`, `"@types/ws": "^8.18.1"` devDependency, `"@agent-talkie/protocol": "workspace:*"`, `"@agent-talkie/persistence": "workspace:*"`, `"typescript"`, `"tsup"`, `"vitest"` aligned with sibling packages.
