**Client→relay mutating control ops use full `Envelope` with `kind: 'control'` and required `idempotencyKey` (UUID):**
- Join: `type: 'space.join'`, `payload: { slug: string }`, `sessionId` must equal bound session, no `to`, no `spaceId` required on join envelope (space not yet known).
- Leave: `type: 'space.leave'`, `payload: {}`, `idempotencyKey` required.

**Idempotency replay rule:** When `tryRecordIdempotencyKey` returns `{ inserted: false }` for `space.join` or `space.leave`, relay MUST NOT mutate membership again. For `space.join`, respond with `{ "type":"space.joined","spaceId":"<current>","slug":"<normalized slug>" }` derived by: `SELECT s.id, s.slug FROM spaces s JOIN space_memberships m ON m.space_id=s.id WHERE m.session_id=? AND m.left_at IS NULL LIMIT 1` — if no row, send `{ "type":"protocol.error","error":"idempotency_replay_mismatch"}` and close.

**SPACE-02 enforcement:** Before inserting new membership for `sessionId`, query `SELECT space_id FROM space_memberships WHERE session_id=? AND left_at IS NULL` — if returns a row and target join space differs (different `space_id`), send `{ "type":"protocol.error","error":"already_in_space"}` JSON and do not insert.

**Create-or-join slug flow (D-01):** `normalizeSpaceSlug(payload.slug)` → `slugNorm`. Transaction:
1. `tryRecordIdempotencyKey(db, idempotencyKey, sessionId, nowMs)`.
2. If duplicate, handle replay rule above and commit early.
3. `SELECT ... FROM spaces WHERE slug=?` — if `status='active'`, use that `spaceId`.
4. Else if `status='archived' AND expires_at > nowMs`, call `reviveSpaceFromArchived(db, id, nowMs)` then use id.
5. Else if missing or expired row blocks reuse: if expired archived row exists run `DELETE` membership cleanup per repo helper OR create new space — **concrete:** if row exists with `status='expired'` or `archived` with `expires_at <= nowMs`, `INSERT` new `spaces` row with new uuid and **same slug** only if old slug released: simpler approach — `UPDATE spaces SET status='expired' WHERE id=?` then `INSERT` new space with slug; **minimal v1:** if archived past TTL, `INSERT` new space with new id and same slug after deleting old space row (CASCADE deletes memberships/transcript) — implement `deleteSpaceCascade(db, spaceId)` using `DELETE FROM spaces WHERE id=?`.
6. If no active usable row, `insertSpaceWithSlug({ slug: slugNorm, nowMs })`.
7. `insertMembership` or `clearMembershipLeftAt` if rejoin same space after leave — if prior `left_at` set for this pair, `clearMembershipLeftAt`.
8. If `countActiveMembers` was 0 before rejoin and space was archived, revival already handled.

**Last member leave (D-03):** After `markMembershipLeft`, if `countActiveMembers(spaceId)==0`, call `setSpaceArchived(db, spaceId, nowMs, 2592000000)`.

**Routing (RELAY-03):** After validation, for `kind: 'conversation'` or `kind: 'control'` messages **except** `space.join`, `space.leave`, `transcript.query`:
- Require `envelope.spaceId` present and UUID.
- Verify sender has active membership in `envelope.spaceId` via SQL `SELECT 1 FROM space_memberships WHERE space_id=? AND session_id=? AND left_at IS NULL`.
- If `envelope.to` present: verify target session has active membership in same `spaceId`; deliver JSON string of serialized envelope to target socket only.
- If `envelope.to` absent: fan-out to all `session_id` in that space with active membership **excluding** `envelope.sessionId`.

**Sender session binding:** `envelope.sessionId` must equal connection-bound session id; else `{type:'protocol.error',error:'session_mismatch'}`.

**Transcript (D-06, D-08):** After successful authorization for any envelope that is **not** a protocol control type in set `{ 'space.join', 'space.leave', 'transcript.query' }`, call `appendTranscriptEntry` with `envelopeJson: JSON.stringify(envelope)` and `kind` from envelope.

**Catch-up (D-08, D-10):** Immediately after sending `space.joined` ack, schedule `sendTranscriptCatchUp({ db, ws, spaceId })` async: `listTranscriptTailBySeq(db,{spaceId,limit:100})` — constant name `CATCH_UP_DEFAULT_LIMIT = 100` in `packages/relay/src/catch-up.ts` — for each row send a JSON message `{ "type":"transcript.catchup","spaceId":"...", "relaySeq": number, "envelope": <parsed object> }` preserving order ascending by `relaySeq`. **Same bounded catch-up after reconnect:** immediately after a successful `session.resumed` bind (02-02), if `findActiveMembershipForSession(db, sessionId)` returns `{ spaceId, slug }`, schedule the same `sendTranscriptCatchUp` for that `spaceId` with the same `CATCH_UP_DEFAULT_LIMIT` (membership already active — do not re-join; still send tail so D-10 is satisfied).

**Transcript retention (D-04, D-07):** Bounded retention for transcript rows is **independent** of space archival TTL. Implement in relay: after each successful `appendTranscriptEntry` (or batched from the same routing tick), enforce a per-space cap by deleting oldest rows by `relay_seq` when count exceeds `TRANSCRIPT_MAX_ROWS_PER_SPACE` (pick one literal constant in `packages/relay/src/router.ts` or `transcript-prune.ts`, e.g. `50000`, and grep it in acceptance). **Deferred for Phase 2:** row `created_at` TTL, export/archive policy — track as follow-up in backlog or next milestone; the per-space row cap is the concrete v1 bound.

**transcript.query control:** `type: 'transcript.query'`, `payload: { afterSeq?: number, limit?: number }`, cap `limit` to `Math.min(payload.limit ?? 50, 500)` — concrete max `500`. Respond with `{ "type":"transcript.query.result", "entries":[...] }` each entry `{relaySeq, envelope}` from DB order ASC.

**Periodic GC:** `setInterval` every `60000` ms calling `DELETE FROM spaces WHERE status='archived' AND expires_at IS NOT NULL AND expires_at <= ?` then same for orphaned transcripts already CASCADE — optional `pruneExpiredSpaces` in `space-lifecycle.ts`.

**Invalid routing:** missing membership → `{type:'protocol.error',error:'not_in_space'}`.
