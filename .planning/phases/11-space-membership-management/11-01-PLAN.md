## 决策追溯（须落实）

- **D-03** 仅空间所有者可销毁；**D-04** 控制消息 `type: "space.destroy"`，`payload: { slug: string }`；须 `idempotencyKey`（与 join/leave 一致）。
- Slug 校验必须调用 `normalizeSpaceSlug`（来自 `@agent-talkie/persistence`），禁止手写另一套正则。
- 发送者须为 `getSessionById` 判定的人类会话（与 orchestrator 控制消息一致的人类门槛）。
- 成功路径：在事务内 `tryRecordIdempotencyKey` → 解析 slug → 校验 owner → 收集 `left_at IS NULL` 的 `session_id` → 对每一成员 `markMembershipLeft`（或等价批量）→ `deleteSpaceById` → 返回待关闭的 `sessionId[]`；`server` 对列表中每个 id 调用 `ctx.registry.get(id)` 若存在则 `close()`。**锁死成功 wire**：在关闭连接前向**发送者** `ctx.ws` 发送 JSON：`{ "type": "space.destroyed", "slug": "<normalized slug>" }`（与 `space.joined` 形式一致）；随后关闭该空间**所有**成员（含发送者）的 WebSocket。
- 幂等重放：若空间已删除且 idempotency 重放匹配，返回对发送者发送 `space.destroyed`（或仅关闭连接二选一，须在实现中二选一并写入本测试）；**推荐**与 `handleSpaceLeave` 类似：重放返回成功语义且不二次 `deleteSpaceById`。
- Owner 为 `null` 时返回 `not_space_owner`（与 RESEARCH 假设一致）。
