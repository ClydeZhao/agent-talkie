Generation must be **32-char lowercase hex** when **writing** from supervisor after receiving child message (child is source of truth — supervisor writes nothing except what child wrote; actually lock is written by **child daemon** per Plan 01 — supervisor only **reads** lock for liveness). **Correction:** Plan 01 has daemon write lock; supervisor **reads** for ensureRelayRunning. **Stale rules (implement exactly):**
1. Missing file → no running relay from lock perspective → need spawn.
2. Read fails or JSON parse fails → treat as stale → `unlink` lock file (best-effort).
3. `pid` or `port` not finite positive integers, or `generation` not matching `/^[0-9a-f]{32}$/` → stale → unlink.
4. `try { process.kill(pid, 0) } catch` → stale → unlink.
5. Else `GET http://127.0.0.1:${port}/__agent-talkie/v1/health?generation=${encodeURIComponent(generation)}` with `fetch` (global `fetch` in Node 20+):
   - Response status `200` and JSON body parses to object with `ok === true` and `generation === <lock.generation>` → **live**.
   - Any other outcome (ECONNREFUSED, non-200, JSON mismatch) → **stale** → unlink.

**`ensureRelayRunning(opts?: { dataDir?: string; forkTimeoutMs?: number })`:** Returns `Promise<{ port: number; generation: string; pid: number; spawned: boolean }>`:
- Resolve `dataDir` = `opts?.dataDir` ?? `process.env.AGENT_TALKIE_DATA_DIR?.trim()` ?? `envPaths(...).data`.
- If lock **live** → return `{ port, generation, pid, spawned: false }` from lock fields.
- If lock **stale** → unlink, fall through.
- Fork: `import { fork } from "node:child_process"; import { createRequire } from "node:module"; import { fileURLToPath } from "node:url";` — `const require = createRequire(import.meta.url); const daemonEntry = fileURLToPath(require.resolve("@agent-talkie/relay/daemon"));` — `fork(daemonEntry, [], { env: { ...process.env, AGENT_TALKIE_DATA_DIR: dataDir }, stdio: ["ignore", "inherit", "inherit"], execArgv: [] })`.
- Await first `message` event within `forkTimeoutMs` default **`15000`** ms; else `child.kill("SIGKILL")` and throw `Error` with message prefix `relay fork timeout:`.
- Validate message: `typeof m === "object" && m !== null && m.type === "relay.ready" && m.v === 1 && Number.isInteger(m.port) && m.port > 0 && m.port <= 65535 && typeof m.generation === "string" && /^[0-9a-f]{32}$/.test(m.generation) && Number.isInteger(m.pid) && m.pid > 0` — if fail, kill child, throw `Error` with message prefix `invalid relay.ready:`.
- Call `child.disconnect()` if `child.connected`.
- Return `{ port: m.port, generation: m.generation, pid: m.pid, spawned: true }`.

**Export `stopRelay(opts?: { dataDir?: string; signal?: NodeJS.Signals })`:** read lock; if missing, return `{ stopped: false, reason: "no_lock" }`; if stale per rules above, `unlink` and return `{ stopped: false, reason: "stale_lock_removed" }`; else `process.kill(lock.pid, opts?.signal ?? "SIGTERM")` in try/catch; return `{ stopped: true, pid: lock.pid }` on success.

**Export `getRelayStatus(opts?: { dataDir?: string })`:** read lock; if missing `{ running: false, reason: "no_lock" }`; if stale after full check `{ running: false, reason: "stale", ... }`; if live `{ running: true, port, pid, generation }`.

| fetch → loopback relay | Validates generation token; prevents blind trust of PID alone |

| Threat ID | Category | Component | Disposition | Mitigate |

| T-03-02-01 | I | IPC `relay.ready` message | mitigate | Reject unless schema matches exact fields; kill child on invalid message |
| T-03-02-02 | E | `stopRelay` SIGTERM | accept | Local operator only; same as killing any user process |
| T-03-02-03 | T | Stale lock unlink | mitigate | Only unlink when rules classify stale — never on live health success |
