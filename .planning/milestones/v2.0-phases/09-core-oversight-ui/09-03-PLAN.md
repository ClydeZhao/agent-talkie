  <action>
    1. 在 `dashboard-store.ts` 顶部定义常量 `const METADATA_UI_DEBOUNCE_MS = 200`（字面量 200）。

    2. 实现 `applyMetadataPatchFromEnvelope(envelope: import("@agent-talkie/protocol").Envelope): void`：若 `envelope.type !== "metadata.patch"` 则 return。对 `envelope.payload` 调用 `metadataPatchPayloadSchema.safeParse`；失败则 return。

    3. 计算 `targetSessionId = parsed.data.namespace === "profile" && parsed.data.targetSessionId !== undefined ? parsed.data.targetSessionId : envelope.sessionId`。

    4. 在 `roster` Map 中找到 `targetSessionId` 对应行；若不存在则 **创建** 占位行（`displayName` 暂为 `targetSessionId` 前 8 位 + `…`，`isHuman`/`runtime`/`workspaceLabel` 空串，`orchestrator`/`owner` false，progress `idle`，`blockedReason` null）以便后续快照刷新可覆盖。

    5. 若 `namespace === "profile"`：合并 `role`、`focus`（仅对 payload.patch 中出现的键赋值）。

    6. 若 `namespace === "status"`：合并 `progress`、`blockedReason`（`blockedReason` 可置 `null` 若 patch 显式清除——按 schema 允许则处理 optional；若仅 undefined 则跳过）。

    7. debounce：每次元数据变更重置 `setTimeout`，**延迟 `METADATA_UI_DEBOUNCE_MS` ms** 后调用已有的 `notifyListeners` / `requestUpdate` 回调；不要在每次 patch 同步触发 UI。

    8. 在 `hydrateFromSpaceSummary` 开始时 `clearTimeout` 该 debounce 定时器句柄（若存在），避免泄漏。
