    - `transcriptFilterSenderSessionId: string | null = null`（`null` = 任何寄件人）
    - `transcriptFilterKind: "all" | "control" | "conversation" = "all"`
    - `transcriptTimeFilter: { kind: "all" } | { kind: "preset"; preset: "5m" | "30m" } | { kind: "custom"; startMs: number; endMs: number }`（初始 `{ kind: "all" }`）

    - `transcriptFilterSenderSessionId`：若非 `null`，要求 `line.envelope.sessionId === transcriptFilterSenderSessionId`
    - `transcriptFilterKind`：`all` 跳过；否则 `line.envelope.kind === transcriptFilterKind`
    - `transcriptTimeFilter`：`all` 跳过；`preset` 以 `Date.now()` 为「现在」、窗口为过去 5 或 30 **分钟**（`5 * 60 * 1000` / `30 * 60 * 1000`）；`custom` 要求 `line.receivedAtMs` 在 `[startMs, endMs]` 闭区间
    5. 实现 `getVisibleTranscriptLines(): TranscriptLine[]`（D-09）:

    - 若 `transcriptSearchQuery.trim() === ""`，返回该数组
    - 否则 `miniSearch.search(transcriptSearchQuery.trim(), { filter: (r) => { const id = r.id; const line = lineByDedupeKey.get(id); return line !== undefined && lineMatchesTranscriptFilters(line); } })`（需维护 `lineByDedupeKey` 从当前 `transcriptLines` 或由 filter 后集合构建），**最终顺序**为原 `transcriptLines` 中顺序、且同时在命中 id 集合内
    6. 导出/公开 `getTranscriptSearchHitIdsForTestOnly` **不要**；改用测试通过 `getVisibleTranscriptLines` 与 `store.transcriptSearchQuery` 断言即可。
