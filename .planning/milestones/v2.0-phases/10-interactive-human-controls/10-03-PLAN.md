## 幂等语义（写死）

- **Fresh**：在 **单个** `db.transaction` 内：先 `INSERT` 新 `idempotency_key` 行（含 `conversation_envelope_id = envelope.id`、`conversation_replay_wire = wire`），若 `changes > 0` 则立即调用调用方传入的 `append: () => void`（内部为 `appendTranscriptEntry` + `pruneTranscriptIfOverCap`），再 `commit`；事务外继续既有 fan-out + 10-01 人类 echo。
- **Replay**：`idempotency_key` 已存在且 `session_id` 与当前 `envelope.sessionId` 一致且 `conversation_envelope_id === envelope.id`：不 append、不 fan-out；仅 `senderWs.send(conversation_replay_wire)`（原始 UTF-8 JSON 字符串，**不要**二次 `JSON.stringify`）。
- **Mismatch**：同 key 已存在但 `conversation_envelope_id` 与当前 `envelope.id` 不一致，或 `conversation_replay_wire` 缺失，或 `session_id` 不一致：`sendJson(senderWs, { type: "protocol.error", error: "idempotency_replay_mismatch" })` 并 `return`。
- **无 `idempotencyKey`**：`conversation` 走现有路径（不启用本门控），兼容非 dashboard 客户端。

## `wire` 定义

`const wire = JSON.stringify(envelope)`，在 `routeEnvelope` 中于 membership 校验通过之后尽早计算，供 insert 与 echo 共用。
