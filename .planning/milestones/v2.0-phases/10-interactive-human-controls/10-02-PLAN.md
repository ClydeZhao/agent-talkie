## Relay 线协议形状（写入 zod 须一致）

来自 `collaboration-handlers.ts` / `fanOutOrchestratorUpdate`：

- `{ type: "orchestrator.designated", spaceId, orchestratorSessionId }`（`orchestratorSessionId` 为 string uuid）
- `{ type: "orchestrator.cleared", spaceId }`
- `{ type: "collaboration.orchestrator", spaceId, orchestratorSessionId }`（`orchestratorSessionId` 可为 `null`）

## 控制信封（sendEnvelope 写死）

使用 `safeParseEnvelope` 前用 Zod 构造 payload：

- `kind: "control"`
- `type: "orchestrator.designate"`：`payload` 满足 `orchestratorDesignatePayloadSchema`（import `@agent-talkie/protocol`），字段为 `{ orchestratorSessionId: string }`。
- `type: "orchestrator.clear"`：`payload` 满足 `orchestratorClearPayloadSchema`（通常为 `{}`）。
- `idempotencyKey: crypto.randomUUID()`（**每个菜单点击各生成新的 UUID**，符合「每次操作独立幂等键」）。
- `version` / `sessionId` / `spaceId` 与 10-01 相同来源；`id: crypto.randomUUID()`。
